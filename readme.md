# Requisitos do Sistema — Cadastro de Usuários para Academia

## Requisitos Funcionais

| ID | Requisito | Descrição |
|---|---|---|
| RF01 | Cadastro de Usuários | O sistema deve permitir o cadastro de alunos com nome, CPF, telefone, endereço, e plano |
| RF02 | Login Administrativo | O sistema deve permitir login de administradores utilizando e-mail e senha. |
| RF03 | Gerenciamento de Usuários | O administrador deve poder cadastrar, editar, visualizar e excluir alunos. |
| RF04 | Controle de Planos | O sistema deve permitir cadastro e gerenciamento dos planos da academia. |
| RF05 | Controle de Pagamentos | O sistema deve registrar pagamentos e informar status de mensalidades. |
| RF05 | Controle de Entrada | O sistema deve registrar data e horário de entrada dos alunos. |
| RF06 | Dashboard Administrativo | O sistema deve apresentar relatórios e informações gerais da academia. |
| RF07 | Bloqueio de Acesso | O sistema deve impedir acesso de alunos inadimplentes ou com plano vencido. |

---

## Requisitos Não Funcionais

| ID | Requisito | Descrição |
|---|---|---|
| RNF01 | Segurança | Os dados biométricos devem ser armazenados de forma criptografada e segura.  |
| RNF02 | Usabilidade | O sistema deve possuir interface simples e intuitiva. |
| RNF03 | Banco de Dados | O sistema deve utilizar banco de dados para armazenamento persistente. |
| RNF04 | Compatibilidade | O sistema deve ser compatível com leitores biométricos e navegadores modernos. |
| RNF05 | Backup | O sistema deve permitir backup periódico dos dados. |
| RNF06 | Manutenção | O código deve ser organizado e documentado para facilitar manutenção futura. |
